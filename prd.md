1. 产品定位
核心目标

帮助盲人在生活场景中完成三类任务：

场景	用户目标	Agent 输出
抓取物品	找到水杯、钥匙、药盒、门把手	“杯子在正前方偏右，约 35 厘米，手再向右一点。”
行走避障	识别椅子、桌角、台阶、玻璃门、低矮障碍物	“停下。正前方半步有椅腿。”
OCR 读文字	读药品、菜单、门牌、电梯按钮、包装标签	“这是矿泉水。标签上写：纯净水，500 毫升。”

现有产品也基本沿着这个方向发展：Google Lookout 包含文本、文档、环境探索、货币、食品标签、找物体、图片描述等模式；Envision App 也支持读文字、描述场景、找物、识别人和条码等能力。

2. 关键设计原则

第一原则：不要描述世界，要指导行动。

普通视觉模型可能说：“桌子上有一个杯子、一本书和一个盘子。”
盲人真正需要的是：“杯子在桌面上，正前方偏右 30 厘米，杯口朝上，可以伸手拿。”

第二原则：安全风险优先于信息完整性。

走路场景下，不要长篇描述环境。Agent 应该只播报对行动有影响的内容：

“停下。”
“前方有台阶。”
“左侧可通过。”
“右前方有人靠近。”

Apple 对类似视觉识别功能也明确提醒：不应在可能受伤、高风险、导航等场景中过度依赖视觉识别功能；你的产品也应该保留同样的安全边界，不能宣称替代盲杖、导盲犬或人工陪同。

第三原则：多模态输出，而不是只靠语音。

W3C 的 XR 无障碍需求也强调多模态支持、输入输出同步和个性化设置。对盲人来说，语音、震动、空间音频、按键、手势、耳机都应该配合使用。

4. 核心模块设计
A. OCR 模块：读文字

OCR 不只是“识别出文字”，还要解决盲人使用时最常见的问题：

问题	Agent 应该怎么做
画面没对准	“文字在画面左侧，请手机向左一点。”
太远	“请靠近一点。”
反光	“有反光，请稍微倾斜包装。”
字太多	先摘要，再问是否全文朗读
药品 / 食品	优先读名称、剂量、保质期、过敏源

技术上，移动端可以先用本地 OCR。比如 Google ML Kit Text Recognition v2 支持中文、日文、韩文、拉丁文等脚本，能识别文本块、行、词、符号，并支持实时识别。

建议输出分三层：

极速摘要：
“这是感冒药，盒子上写每片 500 毫克。”
关键信息：
“有效期：2027 年 3 月。每日三次。”
全文朗读：
用户说“全文”后再读。
B. 物体识别模块：找杯子、钥匙、药盒

这里不能只做分类，还要做空间定位。

识别结果应该包含：

{
  "object": "水杯",
  "confidence": 0.91,
  "position": "正前方偏右",
  "distance_cm": 38,
  "orientation": "杯口朝上",
  "reachable": true,
  "risk": "低"
}

对“抓水杯喝水”这个场景，Agent 流程可以这样：

用户：帮我找水杯
Agent：开始寻找水杯
摄像头扫描
Agent：发现水杯，在正前方偏右，约 40 厘米
用户伸手
Agent：手再向右一点
Agent：再往前 10 厘米
Agent：已经接近杯身，慢一点
Agent：可以抓取

如果检测到杯子可能装有液体，提示要更谨慎：

“杯子可能有水，建议握住杯身中部，慢慢拿起。”

C. 障碍物识别模块：行走辅助

这是最危险的部分，必须单独设计，不要和普通场景描述混在一起。

风险分级
等级	条件	输出
P0 立即危险	0.5 米内有障碍、台阶、车辆、坑洞	“停下！”
P1 高风险	1 米内有椅子、桌角、门槛、行人靠近	“前方一步有障碍。”
P2 中风险	2 米内有可绕行物体	“右前方有椅子，偏左走。”
P3 信息提示	门、楼梯、电梯、路牌	“左侧是电梯门。”
输出要短

错误示范：

“前方有一个木质椅子，旁边有一张白色桌子，地面是灰色的。”

正确示范：

“停下。正前方有椅腿。”
“左侧可通过。”
“前方三步是门。”

深度信息很关键

普通单目摄像头只能“看见”，但不一定能准确判断距离。建议优先使用带深度能力的设备，例如 iPhone LiDAR、Android ToF、双目摄像头，或 ARCore Depth API。ARCore Depth API 可以让设备通过深度图理解场景中真实物体的大小和形状，用于距离、遮挡和空间交互判断。

5. Agent 的工作模式

建议设计 5 个模式，而不是一个大杂烩模式。

1）找物模式

用户说：

“帮我找杯子。”
“我的钥匙在哪？”
“找门把手。”

Agent 只关注目标物，不播报无关信息。

输出：

“杯子在右前方，距离大约半臂长。”
“继续向右扫。”
“找到了，在桌边，请小心边缘。”

2）抓取模式

用于水杯、药瓶、筷子、手机、门把手。

需要识别：

目标物体
物体边界
桌面高度
用户手的位置
目标和手的相对距离
是否存在危险：热水、刀具、桌边、玻璃杯

输出：

“手向前。”
“向左一点。”
“慢一点。”
“已经碰到杯身。”

3）行走模式

只播报障碍和方向。

默认静默，危险时打断。

输出优先级：

“停下”
“向左 / 向右”
“前方有台阶 / 门 / 人”
“可以继续”
4）读文字模式

适合菜单、包装、药品、账单、门牌、电梯按钮。

输出：

“检测到文字。”
“这是 7 楼按钮。”
“药盒上写：布洛芬缓释胶囊。”

Google Lookout 的文档模式也采用了类似的实时引导方式：用户移动设备时，系统会给出实时引导以拍下完整文档。

5）问答模式

用户可以问：

“这是什么？”
“这里有没有空座位？”
“哪个是矿泉水？”
“前面能不能走？”
“这个杯子满了吗？”

Agent 需要把视觉结果、深度结果和 OCR 结果融合后回答。

7. 关键体验设计
语音输出规则

不要说：

“我看到了一个白色圆柱形杯子，它位于图像右下角。”

要说：

“杯子在右前方，手向右一点。”

方向表达规则

用盲人容易理解的表达：

视觉坐标	语音表达
图像左侧	“左前方”
图像中心	“正前方”
图像右侧	“右前方”
近距离	“半步内 / 一臂内”
中距离	“一步左右”
远距离	“三步外”
震动编码
震动方式	含义
左侧短震	向左
右侧短震	向右
双侧急震	停下
慢速连续震	接近目标
快速连续震	危险靠近
8. 安全机制

这部分非常重要。

必须有“不确定性表达”

当模型不确定时，不要强行给结论。

错误：

“可以走。”

正确：

“前方画面不清楚，建议停下重新扫描。”

必须有失败状态

常见失败场景：

光线太暗
摄像头被遮挡
反光严重
快速运动导致模糊
人群密集
下雨 / 夜间 / 玻璃门
透明障碍物
地面坑洞
台阶边缘
悬挂障碍物

对应输出：

“画面太暗，无法可靠识别。”
“前方不确定，请使用盲杖确认。”
“检测到可能有玻璃门，请慢行。”

不能替代的东西

产品说明中要明确：

不替代盲杖
不替代导盲犬
不保证导航安全
不用于过马路决策
不用于医疗诊断
不对危险环境做绝对判断
9. 推荐的 MVP 功能范围

第一版不要做太大，建议只做 4 个高频场景。

MVP 1：找杯子并辅助抓取

能力：

识别杯子 / 水瓶
判断左中右
判断距离
引导手靠近
提醒桌边和倾倒风险
MVP 2：读包装和药盒

能力：

OCR 识别
自动摘要
读药名、剂量、有效期
识别食品标签和条码
MVP 3：室内避障

能力：

椅子、桌子、门、墙、台阶
近距离障碍提示
左右绕行建议
光线不足提醒
MVP 4：找门和电梯按钮

能力：

识别门、门把手、电梯、按钮
OCR 读楼层数字
指导用户按按钮
10. Agent 决策逻辑示例

以“抓水杯”为例：

1. 用户说：找水杯
2. Agent 进入找物模式
3. 摄像头扫描环境
4. 检测到多个物体：杯子、盘子、手机
5. 过滤目标：杯子
6. 估计距离和方向
7. 判断是否可触达
8. 输出：杯子在右前方，约 40 厘米
9. 检测用户手部位置
10. 实时引导：向右、向前、慢一点
11. 检测手接近杯身
12. 输出：可以抓取
13. 抓取后提醒：杯子可能有水，请保持水平

以“走路避障”为例：

1. Agent 进入行走模式
2. 每帧检测前方 0–2 米区域
3. 优先识别地面、台阶、椅腿、墙、行人
4. 计算危险等级
5. 如果 P0：立即打断语音
6. 如果 P1：短提示
7. 如果 P2：给绕行方向
8. 如果无风险：保持静默

12. 评估指标

不要只看 mAP、OCR 准确率这种模型指标，还要看用户任务成功率。

指标	说明
抓杯成功率	用户是否能安全抓到杯子
平均抓取时间	从开始寻找到成功抓取
危险漏报率	最重要，越低越好
误报打扰率	太多误报会让用户不信任系统
指令可理解率	用户听完是否知道怎么做
OCR 有效信息召回率	药名、剂量、有效期是否读对
延迟	行走避障必须低延迟
用户信任度	是否愿意在日常继续使用

其中最重要的是：

危险漏报率 > 指令可执行性 > 延迟 > 模型准确率。

13. 推荐产品形态
最现实的第一版

手机 App + 胸前挂绳 / 肩带固定。

优点：

成本低
易开发
摄像头质量好
用户容易试用

缺点：

视角不稳定
手持时不方便
行走场景体验有限
更理想的版本

智能眼镜 + 开放式耳机 + 震动反馈。

优点：

解放双手
第一视角自然
更适合走路和抓取
可以用空间音频提示方向

缺点：

成本更高
续航压力大
隐私风险更高
14. 我建议的产品命名与一句话描述

可以叫：

GuideVision / 伴行眼 / SeeMate / 触景助手

一句话描述：

一款为盲人设计的实时视觉辅助 Agent，把物体、文字和障碍物转化为可执行的语音与震动指令，帮助用户找物、读字、抓取和避障。

15. 最小可行原型建议

第一阶段只做这条闭环：

“找水杯 → 定位水杯 → 引导手靠近 → 提醒抓取”

因为它能验证最核心的能力：

物体识别
距离估计
方向表达
实时反馈
手部追踪
安全提示
用户信任

做好这个闭环后，再扩展到：

读药盒
找门把手
电梯按钮识别
室内避障
户外低速辅助行走

最终形态不是“一个会看图的 OCR 工具”，而是一个行动型视觉 Agent：它不追求说得多，而是追求在正确时间说出最有用、最安全的一句话。
